      Compatible Firmware

      Next steps

      Firmware installed successfully. Finish setup by provisioning Wi‑Fi and assigning a friendly device name.

      1. Provision Wi‑Fi

        Keep the hub connected and start the Improv Wi‑Fi workflow so it can join your network:

        Open Improv Wi‑Fi

        Common pitfall: Improv only works on 2.4 GHz SSIDs without captive portals, VPN requirements, or MAC filtering. If provisioning stalls, double-check the network and leave BOOT/RESET alone until the flow finishes.

      2. Name the device

        Apply the fleet naming convention before installation day. Use this template with the Sense360 CLI or Fleet Manager rename dialog.

        sense360-cli device name --id <device-id> --name "Building-Room-Device"

        Common pitfall: Names with spaces, emojis, or punctuation are rejected. Stick to letters, numbers, and hyphens—for example Lobby-01-East.

      Rescue Install

      If the recommended firmware fails to boot or the hub is stuck in a flashing loop, use the rescue image to restore the bootloader before trying again.

      Sense360 Rescue Firmware
      Rescue Build
      v1.0.0-rescue 31 B Jan 1, 2024

      Launches a known-good recovery image that skips configuration matching to bring unresponsive devices back online.

      Always available for emergency recovery.

      After the install finishes, keep the hub powered on and connected to the same network as Home Assistant. Discovery usually happens within a minute. If the device doesn’t appear, open the integrations dashboard and follow the ESPHome adoption steps to add it manually.
